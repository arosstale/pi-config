Algorithmic philosophies are expressed through code. Output includes a .md (philosophy), .html (viewer), and .js (algorithms). The process consists of two steps: Algorithmic Philosophy Creation and p5.js Implementation.

## ALGORITHMIC PHILOSOPHY CREATION

Create an ALGORITHMIC PHILOSOPHY to be interpreted through code. This philosophy informs the p5.js implementation.

**Use when:** The request explicitly asks for “generative art,” “algorithmic art,” or similar terms, or when the request describes visual concepts suitable for procedural generation (e.g., “flow fields,” “particle systems,” “organic forms”).

### THE CRITICAL UNDERSTANDING
- **Input:** Subtle artistic direction; a starting point, *not* a rigid specification.
- **Output:** A concise algorithmic philosophy/generative aesthetic movement document (.md).
- **Next:** The same version implements the philosophy with p5.js – 90% algorithmic, 10% essential, *interactive* parameters.

The philosophy should be written as a manifesto followed by a description of the implementing algorithm.

The philosophy *must* emphasize: Algorithmic expression, emergent behavior, computational beauty, and seeded variation.

### HOW TO GENERATE AN ALGORITHMIC PHILOSOPHY

**1. Name the movement** (1-2 words): Examples: "Organic Turbulence", "Quantum Harmonics", "Emergent Stillness".

**2. Articulate the philosophy** (4-6 paragraphs - concise). Focus on *why* the aesthetic is compelling, not just *what* it looks like.

Express how this manifests through:
- **Computational processes & mathematical relationships:** How are algorithms and math central to the aesthetic? (e.g., "The aesthetic relies on the inherent beauty of the Fourier transform, revealing hidden structures in complex data.")
- **Noise functions & randomness patterns:** What role does randomness play? (e.g., "Perlin noise is used to simulate natural, unpredictable variations, creating a sense of organic growth.")
- **Particle behaviors & field dynamics:** How do particles and forces interact? (e.g., "Particles are treated as agents responding to attractive and repulsive forces, creating dynamic patterns of aggregation and dispersion.")
- **Temporal evolution & system states:** How does the art change over time? (e.g., “The system evolves towards a state of equilibrium, but is perpetually disrupted by external noise, preventing stagnation.”)
- **Parametric variation & emergent complexity:** How can parameters be adjusted to create diverse outcomes? (e.g., "Small changes in seed values lead to dramatically different visual results, highlighting the sensitivity of the system.")

**3. Algorithmic Expression (Implementation Sketch):** Briefly describe the core algorithm. (2-4 sentences)

**CRITICAL GUIDELINES:**
- **Avoid redundancy**: Mention algorithmic aspects only once per concept.
- **Emphasize craftsmanship**: The philosophy MUST repeatedly stress a meticulously crafted algorithm, the product of deep expertise, painstaking optimization, and master-level implementation. Use phrases like "meticulously tuned," "carefully calibrated," "expertly implemented," and "refined through iterative development."
- **Leave creative space**: Be specific algorithmically, but concise enough for interpretive implementation. Don't fully define the implementation; leave room for artistic interpretation.
- **Interactive Parameters**: Identify 2-3 core parameters that a user should be able to control (e.g., noise scale, particle count, attraction radius).
